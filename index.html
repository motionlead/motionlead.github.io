MotionLead iOS documentation

Getting started

The MotionLead framework interface is composed of two classes :
* MLManager : best suited for apps built with UIKit. Once launched, the MLManager is entirely autonomous and will display rich ads on your application when one is available. It provide less control but is really easy to integrate.
* MLView : it is a subclass of UIView and it allows you to display ads on specific screens of your app. It provide you more control on how the ad will be displayed.

Installation

This tutorial will show you how to install the MotionLead framework to an existing project.

Step 1: Download the framework

You can download the latest release on the framework on your MotionLead control panel

Step 2: Add the framework to your project

Drag and drop the framework to your project

Make sure to select Copy items into destination group's folder (if needed)

Step 3: Add the dependencies

Click on your application name then click on Build Phases. In the Link Binary With Libraries add the following frameworks :

  • libz.dylib
  • QuartzCore.framework
  • AdSupport.framework
  • StoreKit.framework
  • CoreTelephony.framework
  • SystemConfiguration.framework

Step 3.5: Pre iOS6 support

If your app need to be compatible with iOS versions prior to 6.0 you have to set AdSupport.framework to optional

Step 4: Modify the Build Settings

Click on your application name then click on Build Settings. Select the All tab, then search for Other Linker Flags. Double click on it and add the flag -ObjC. It will force Xcode to compile the categories provided by the framework.

Step 5: Build

Build your app to make sure that your project is configured correctly.

MLManager

Using the MLManager is the simplest way to add the MotionLead technology to your app, however it is not adapted to all kind of apps. If you do not need control on how and when the ad is displayed, the MLManager is for you, otherwise take a look at MLView.

Usage

In your app delegate implementation file, add the following import at the top

#import <MotionLead/MotionLead.h>

Then, in the application:didFinishLaunchingWithOptions: method add the following line by replacing YOUR_PUBLISHER_KEY by the key provided by MotionLead

[MLManager startWithKey:@"YOUR_PUBLISHER_KEY"];

Congratulation, MotionLead is now up and running on your app! If you want to tune the behaviour of the MLManager go take a look at MLManager.h.

MLView

The MLView gives you more control on how and where the ad will be displayed.

Usage

The MLView can be created as a 300x250 unit or as a fullscreen ad (for both the iPhone and iPad).

300x250 ad unit

  1. Import the MotionLead framework

     #import <MotionLead/MotionLead.h>>

  2. Create a property holding the MLView instance

     @interface ViewController : UIViewController
 @property (strong, nonatomic)   MLView  *adView;
 @end

  3. Initialize self.adView, for instance in the viewDidLoad method if you are in a UIViewController. Replace YOUR_PUBLISHER_KEY by the key provided by MotionLead

     self.adView = [[MLView alloc] initWithPublisherKey:@"YOUR_PUBLISHER_KEY" frame:MLVIEW_300x250_FRAME];

  4. Optional: change the origin of self.adView

     CGRect frame = self.adView.frame;
 frame.origin.x = 100;
 frame.origin.y = 200;
 self.adView.frame = frame;

  5. Add self.adView to the view hierarchie

     [self.view addSubview:self.adView];

  6. Start loading the ad

     [self.adView loadAd];

  7. When you need to display the ad, call:

     [self.adView displayAd];

Interstitial ad unit

  1. Import the MotionLead framework

     #import <MotionLead/MotionLead.h>>

  2. Create a property holding the MLView instance

     @interface ViewController : UIViewController
 @property (strong, nonatomic)   MLView  *adView;
 @end

  3. Initialize self.adView, for instance in the viewDidLoad method if you are in a UIViewController. Replace YOUR_PUBLISHER_KEY by the key provided by MotionLead

     self.adView = [[MLView alloc] initWithPublisherKey:@"YOUR_PUBLISHER_KEY" frame:MLVIEW_INTERSTITIAL_FRAME];

  4. Start loading the ad

     [self.adView loadAd];

  5. When you need to display the ad, call:

     [self.adView displayAd];

    The adView will be displayed in the main UIWindow of your app without blocking the UI.

  6. When you will need to remove the ad, call:

     [self.adView removeFromSuperview];

MLView delegate

You should register to your MLView delegate in order to respond to changes in the state of the ad.

In order to register to the delegate, add the protocol MLViewDelegate to your interface

@interface ViewController : UIViewController <MLViewDelegate>

Then, after initializing your MLView write :

self.adView.delegate = self;

You can now implements the methods defined in the MLViewDelegate protocol defined in the file MLView.h

Examples

To display the ad right after it has been downloaded, write the following in your class:

- (void)mlViewDidLoadAd:(MLView *)view {
    [view displayAd];
}

To download a new ad right after the previous ad finished running:

- (void)mlViewDidFinishAd:(MLView *)view {
    [view loadAd];
}
